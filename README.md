# Online Courses Catalog - Тестовое задание


Проект представляет собой каталог курсов, реализованный на **Vanilla JS** и **SCSS** вместе с **БЭМ** в рамках тестового задания.

**Демо** - https://test-task-layout-vanillajs.vercel.app/



## Техническое задание

- Pixel-perfect вёрстка по макету.
- Адаптив «резиновый» (без жёстких брейкпоинтов, плавно масштабируется).
- Фильтрация по категориям (всё сгруппировано по категориям).
- Чистый и понятный код по [https://ru.bem.info/methodology/](https://ru.bem.info/methodology/).
- Стек: HTML + JavaScript (vanilla) + SCSS. Никаких библиотек и фреймворков — только нативный JS.



## Технологический стек

- **Build Tool**: [Vite](https://main.vite.dev/) 7.2 (выбран за мгновенный HMR и эффективный ролл-ап бандлинг).
- **Styles**: SCSS (Dart Sass) + [Lightning CSS](https://lightningcss.dev/) (для продвинутой минификации и автопрефиксов).
- **Vanilla JS**



## Что уже есть помимо того, что описано в ТЗ

- Возможность настроить поддержку браузеров в файле `.browserslistrc`. вместо указания версий браузеров можно использовать `baseline widely available`.
- **Images**: 
  - Внедрена стратегия адаптивного приоритета. Первые 9 карточек (Above-the-fold) имеют `fetchpriority="high"` и `loading="eager"`. Остальные — нативный `lazy-loading`. 
  - Используется формат `avif`, доступный на всех основных браузерах.
  - Атрибут `decoding="async"` для всех изображений карточек позволяет браузеру декодировать контент вне основного потока.
- **Fonts**: 
  - Использование формата `woff2`. 
  - Настроен `preload` для всех начертаний, используемых на первом экране. 
  - Применен `font-display: swap` для предотвращения невидимого текста (FOIT).
  - Адаптивная типографика: Реализована через `clamp()`.
- **JS**:
  - Deep Linking: Синхронизация состояния фильтров с `URLSearchParams`. Это позволяет сохранять состояние поиска при перезагрузке страницы и делиться ссылками.
  - Отказ от тяжелого `innerHTML` в пользу `<template>`. Это минимизирует нагрузку на парсер и защищает от XSS.
  - Использование `DocumentFragment` при массовом обновлении сетки, что сокращает количество операций Reflow/Repaint.
  - Использование `debounce` для поиска, чтобы не нагружать систему при каждом нажатии клавиши.
